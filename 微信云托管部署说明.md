# 微信云托管部署说明

## 一、后端配置（已完成）

### 1. application.yml - 使用 H2 内存数据库

```yaml
spring:
  datasource:
    driver-class-name: org.h2.Driver
    url: jdbc:h2:mem:campus_express;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE;MODE=MySQL
    username: sa
    password:
  
  h2:
    console:
      enabled: true
      path: /h2-console
  
  sql:
    init:
      mode: always
      schema-locations: classpath:db/schema.sql
      data-locations: classpath:db/data.sql
```

**说明**：
- 使用 H2 内存数据库，无需外部数据库依赖
- 启动时自动执行 schema.sql 和 data.sql
- H2 控制台地址：`http://localhost:8080/h2-console`
- JDBC URL：`jdbc:h2:mem:campus_express`

---

### 2. Dockerfile - 多阶段构建

```dockerfile
# 阶段1：编译
FROM maven:3.8.7-openjdk-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
RUN mvn clean package -DskipTests

# 阶段2：运行
FROM openjdk:17-jre-slim
WORKDIR /app
COPY --from=build /app/target/*.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]
```

**说明**：
- 使用 Java 17（与 pom.xml 配置一致）
- 多阶段构建，减小镜像体积
- 自动编译打包，无需本地构建

---

### 3. pom.xml - Java 版本修改

```xml
<properties>
    <java.version>17</java.version>
    ...
</properties>
```

**说明**：
- 从 Java 1.8 升级到 Java 17
- 与 Dockerfile 中的 openjdk:17-jre-slim 匹配

---

### 4. schema.sql - H2 兼容语法

**修改内容**：
- `IDENTITY(1,1)` → `AUTO_INCREMENT`
- `GETDATE()` → `CURRENT_TIMESTAMP()`
- `BIT` → `BOOLEAN`
- `DATETIME` → `TIMESTAMP`
- 去掉方括号 `[]`
- 去掉 SQL Server 特有的 `IF NOT EXISTS` 检查

**示例**：
```sql
-- SQL Server
CREATE TABLE [user] (
  [id] BIGINT IDENTITY(1,1) PRIMARY KEY,
  [create_time] DATETIME DEFAULT GETDATE()
);

-- H2
CREATE TABLE IF NOT EXISTS user (
  id BIGINT AUTO_INCREMENT PRIMARY KEY,
  create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP()
);
```

## 三、部署步骤

### 步骤 1：本地测试

1. **启动后端**：

   - 测试登录、绑定手机号等功能

---

### 步骤 2：部署到微信云托管

#### 2.1 创建云托管服务

1. 登录微信公众平台
2. 进入「云开发」→「云托管」
3. 点击「新建服务」
4. 选择「容器服务」

#### 2.2 上传代码

**方式 1：代码库部署**
1. 将代码推送到 GitHub/GitLab
2. 在云托管中选择「代码库部署」
3. 选择代码库和分支
4. 系统自动构建镜像

**方式 2：镜像部署**
1. 本地构建镜像：
   ```bash
   cd D:\NewDelivery\campus-express-backend
   docker build -t campus-express-backend .
   ```
2. 推送到镜像仓库（如腾讯云容器镜像服务）
3. 在云托管中选择「镜像部署」
4. 输入镜像地址

#### 2.3 配置服务

- **端口**：8080
- **环境变量**：（可选）
  - `OBS_ENABLED=false`
- **扩缩容配置**：
  - 最小实例：0（节省资源）
  - 最大实例：1

#### 2.4 获取访问地址

部署成功后，云托管会提供一个访问地址，格式如：
```
https://xxx.ap-shanghai.run.tcloudbase.com
```

## 五、常见问题

### Q1: 部署后无法访问

**原因**：端口配置错误或服务未启动

**解决**：
1. 检查 Dockerfile 中的 EXPOSE 8080
2. 检查云托管配置的端口是否为 8080
3. 查看云托管日志

---

### Q2: 数据库初始化失败

**原因**：schema.sql 语法不兼容 H2

**解决**：
1. 检查 schema.sql 是否使用 H2 兼容语法
2. 查看 H2 控制台确认表是否创建成功
3. 查看应用启动日志

---

### Q3: 小程序请求失败

**原因**：baseUrl 配置错误或域名未配置

**解决**：
1. 确认 baseUrl 是否为云托管地址
2. 在微信公众平台配置服务器域名
   - 登录微信公众平台
   - 开发管理 → 开发设置 → 服务器域名
   - 添加云托管域名

---

### Q4: 微信登录失败

**原因**：appid 或 secret 配置错误

**解决**：
1. 检查 application.yml 中的 wechat.appid 和 wechat.secret
2. 确认小程序已上线或使用测试号

---

## 六、优化建议

### 1. 数据持久化

H2 内存数据库重启后数据会丢失。如需持久化：

**方案 1：使用云托管数据库**
- 微信云托管提供 MySQL 数据库
- 修改 application.yml 连接云托管 MySQL

**方案 2：使用外部数据库**
- 腾讯云 MySQL
- 阿里云 RDS
- 修改 application.yml 连接外部数据库

---

### 2. 环境变量配置

将敏感信息改为环境变量：

```yaml
wechat:
  appid: ${WECHAT_APPID:wxf75e60b7eab2499b}
  secret: ${WECHAT_SECRET:c7ae0fb7819190c6f8e019afd34b9379}

jwt:
  secret: ${JWT_SECRET:campus-express-secret-key-2024-!@#abcXYZ}
```

在云托管中配置环境变量：
- `WECHAT_APPID`
- `WECHAT_SECRET`
- `JWT_SECRET`

---

### 3. 日志监控

- 使用云托管的日志服务查看应用日志
- 配置告警规则（如错误率过高）
